# Executive Summary

This project was undertaken in collaboration with mDetect Pty Ltd to develop a GPS-denied localisation and navigation platform for a mobile robotic system intended for operation in underground and confined environments. The motivation for the project arose from the limitations of Global Positioning System (GPS) technology in tunnels, chambers, mines, and interconnected voids, where reliable position information is required for future deployment of autonomous muon detector systems.

The primary objective of the project was to develop a robotic platform capable of estimating its position and orientation without relying on external positioning infrastructure. To achieve this, a localisation system was developed using wheel encoder odometry and an MPU6050 Inertial Measurement Unit (IMU). Encoder measurements were used to estimate robot displacement, while IMU heading information enabled transformation of local movement into global position estimates. The resulting system provided real-time estimation of robot position in a two-dimensional coordinate frame.

Several challenges were encountered throughout development, including motion instability, encoder measurement noise, accumulated odometry errors, and hardware resource limitations. These challenges were addressed through motor scaling, Kalman filtering techniques, and motion control improvements, resulting in improved localisation accuracy and navigation performance.

To support advanced sensing and navigation capabilities, a Raspberry Pi 4 was integrated into the system architecture. Serial communication was established between the Arduino Uno and Raspberry Pi, enabling real-time transmission of localisation data. A Python-based software framework was developed to provide robot monitoring, data logging, and high-level navigation functionality.

The final stage of the project involved integration of a COIN-D6 Time-of-Flight LiDAR sensor. The LiDAR system enabled environmental sensing, obstacle detection, and emergency stop functionality. Additional navigation features including waypoint management, path planning concepts, and lightweight SLAM foundations were also investigated.

The project successfully achieved its primary objectives and demonstrated the feasibility of combining encoder odometry, IMU heading estimation, Raspberry Pi processing, and LiDAR sensing to create a GPS-free robotic localisation platform. The developed system provides a strong foundation for future autonomous navigation, mapping, and underground sensing applications for mDetect Pty Ltd.

# Introduction

## Background

Underground environments such as tunnels, chambers, mines, and interconnected voids present significant challenges for robotic navigation and localisation. Unlike outdoor environments, Global Positioning System (GPS) signals are unavailable or severely degraded underground, making conventional localisation techniques ineffective. Accurate localisation is essential for autonomous robotic systems operating in these environments, particularly when precise position information is required for inspection, mapping, and scientific data collection.

mDetect Pty Ltd is developing a muon detector platform for underground sensing applications. Muon detectors use naturally occurring cosmic ray muons to investigate subsurface structures and identify hidden voids, geological formations, and underground features. For muon measurements to be meaningful, detector readings must be accurately associated with the robot’s location and orientation at the time of measurement. Consequently, a reliable GPS-free localisation system is required to support future deployment of autonomous muon detection platforms.

![Muon Detector (Mdetect.com.au, 2025)](/images/projects/mdetect/image3.png)

Robotic localisation in GPS-denied environments commonly relies on onboard sensors such as wheel encoders, inertial measurement units (IMUs), cameras, and LiDAR sensors. Each sensing technology offers advantages and limitations. Wheel encoders provide distance travelled information but are affected by wheel slip and accumulated errors. IMUs provide orientation information but are subject to sensor drift over time. LiDAR sensors provide environmental perception and obstacle detection capabilities but require greater computational resources for processing.

To address these challenges, this project investigated the development of a GPS-free localisation platform using a combination of encoder odometry, IMU-based heading estimation, and LiDAR sensing. The system was implemented using an Arduino Uno microcontroller for low-level robot control and a Raspberry Pi 4 for higher-level sensing and navigation tasks.

# Literature Review

## Mobile Robot Localisation Using Odometry

Robot localisation is a fundamental requirement for autonomous navigation systems. Localisation refers to the ability of a robot to estimate its position and orientation within a known or unknown environment. In GPS-denied environments such as tunnels, underground chambers, and mines, localisation becomes particularly challenging because external positioning systems are unavailable.

One of the most widely used localisation techniques is wheel encoder odometry. Odometry estimates robot displacement by measuring wheel rotation and converting encoder counts into travelled distance. The method is computationally efficient and can be implemented using low-cost hardware, making it suitable for embedded robotic systems.

Borenstein and Feng (1996) investigated systematic odometry errors in mobile robots and identified wheel slip, uneven surfaces, and mechanical tolerances as major sources of localisation error. Their work demonstrated that odometry errors accumulate over time, causing increasing deviation between the estimated and actual robot position.

The distance travelled by a wheel can be calculated using:

Distance = (Encoder Counts / Counts per Revolution) × Wheel Circumference

Encoder-based odometry provides a simple and effective method for short-distance localisation. However, due to accumulated errors, odometry alone is generally insufficient for long-term navigation and is often combined with additional sensors.

![Figure 2.1.1 Encoder Odometry Principle (Guo et al., 2025)](/images/projects/mdetect/image7.png)

The findings from previous studies indicate that encoder odometry remains an essential component of mobile robot localisation but requires complementary sensing technologies to improve long-term accuracy.

## Inertial Measurement Units for Navigation

An Inertial Measurement Unit (IMU) is commonly used in robotic systems to estimate orientation, acceleration, and angular velocity. IMUs typically consist of accelerometers and gyroscopes that measure motion in multiple axes.

The MPU6050 sensor was selected for this project due to its low cost, ease of integration, and availability. The sensor combines a three-axis accelerometer and a three-axis gyroscope within a single package, enabling measurement of robot orientation and motion.

Brossard, Barrau, and Bonnabel (2019) investigated IMU-based dead reckoning methods and demonstrated that inertial sensors can provide reliable orientation information for autonomous navigation systems. Their work showed that combining IMU data with other localisation methods can significantly improve navigation performance.

In mobile robot applications, heading estimation is particularly important because orientation errors directly affect position estimation. Even small heading errors can result in large position errors when integrated over long distances.

The primary limitation of IMU systems is sensor drift. Gyroscope bias and measurement noise cause heading errors to accumulate over time. Consequently, IMU sensors are typically integrated with odometry or external sensing systems to improve localisation performance.

For this project, the IMU was used primarily to estimate the robot heading angle, which was then combined with encoder-based displacement measurements to calculate robot position within a two-dimensional coordinate frame.

![Figure 2.2.1 MPU6050 IMU Sensor and Heading Estimation (Last Minute Engineers, 2021)](/images/projects/mdetect/image8.png)

The literature suggests that combining encoder odometry and IMU measurements can significantly improve localisation accuracy compared to using either sensing method independently.

## Proportional Integral Derivative (PID) Control

Motion control is a critical requirement for autonomous robotic systems. Variations in motor performance can cause robots to drift from their intended path, reducing localisation accuracy and navigation reliability.

The Proportional Integral Derivative (PID) controller is one of the most widely used control algorithms in industrial automation and robotics. PID controllers continuously calculate the difference between a desired value and a measured value, generating corrective actions to minimise error.

According to Sundström et al. (2024), PID controllers remain the industry standard for many motion control applications due to their simplicity, reliability, and effectiveness.

The proportional component provides immediate correction based on the current error. The integral component compensates for accumulated errors over time, while the derivative component predicts future behaviour based on the rate of error change.

In mobile robotics, PID control can be used to:

- Maintain straight-line motion.

- Improve turning accuracy.

- Compensate for motor performance differences.

- Reduce navigation errors.

Although PID control improves robot motion stability, careful tuning is required to achieve optimal performance. Excessive gains can cause oscillation, while insufficient gains may result in slow response.

The literature consistently demonstrates that PID control remains an effective method for improving navigation accuracy in differential-drive and mobile robotic platforms.

## Kalman Filtering for Sensor Noise Reduction

Sensor measurements obtained from encoders and IMUs are often affected by noise, vibration, and measurement uncertainty. Noise can significantly reduce localisation accuracy if not properly managed.

The Kalman filter is a mathematical algorithm designed to estimate the true state of a system by combining measurements and predictions. It has become one of the most widely used filtering techniques in robotics and autonomous systems.

Aldibs and Mısır (2025) demonstrated the effectiveness of Kalman filtering for processing LiDAR and localisation data in mobile robots. Their results showed that filtering techniques can reduce measurement noise and improve tracking performance.

In encoder-based localisation systems, Kalman filtering can be applied to smooth pulse measurements before position calculations are performed. This reduces fluctuations caused by wheel vibration, electrical interference, and irregular surface conditions.

Advantages of Kalman filtering include:

- Reduction of measurement noise.

- Improved sensor stability.

- More reliable localisation estimates.

- Enhanced system robustness.

For this project, Kalman filtering was investigated as a method to reduce encoder measurement noise and improve localisation accuracy.

![Figure 2.4.1 Raw Encoder Data versus Kalman Filtered Data](/images/projects/mdetect/image9.png)

(Severin, Dobrea and Dobrea, 2020)

The literature indicates that Kalman filtering can significantly improve sensor quality and is widely adopted in robotic navigation systems.

## LiDAR-Based Navigation and Obstacle Detection

LiDAR (Light Detection and Ranging) has become one of the most important sensing technologies for autonomous robots. LiDAR systems determine the distance to surrounding objects by measuring the time required for laser pulses to return after reflection.

Compared with ultrasonic sensors and cameras, LiDAR provides higher measurement accuracy and greater environmental awareness. As a result, LiDAR has become widely used in autonomous vehicles, robotic mapping systems, and industrial automation.

Alsalamy, Frels, and Danowitz (2018) demonstrated that LiDAR sensors can be effectively used for autonomous navigation and mapping applications. Their work highlighted the ability of LiDAR systems to generate detailed environmental information for path planning and obstacle avoidance.

LiDAR systems provide several advantages:

- Accurate distance measurement.

- Real-time obstacle detection.

- Environmental mapping capabilities.

- Support for localisation and navigation.

However, LiDAR systems require greater computational resources than encoder or IMU systems. Consequently, high-level processing platforms such as Raspberry Pi are often used to manage LiDAR data processing tasks.

For this project, a COIN-D6 Time-of-Flight (ToF) LiDAR sensor was selected to provide obstacle detection and environmental sensing capabilities.

![Figure 2.5.1 COIN-D6 LiDAR Sensor (Witmotion, 2025)](/images/projects/mdetect/image10.png)

The literature demonstrates that LiDAR technology can significantly improve robotic navigation performance, particularly in GPS-denied environments.

## Summary Literature Reviews

The literature review identified encoder odometry, IMU-based heading estimation, PID control, Kalman filtering, and LiDAR sensing as suitable technologies for GPS-free robotic localisation and navigation.

Encoder odometry provides a simple method for estimating robot displacement but suffers from accumulated errors. IMUs provide orientation information but are affected by sensor drift. Kalman filtering can reduce measurement noise, while PID control improves motion stability and navigation accuracy. LiDAR sensors provide environmental awareness and obstacle detection capabilities that support safe autonomous operation.

Based on these findings, a localisation architecture combining wheel encoders, an MPU6050 IMU, Raspberry Pi processing, and COIN-D6 LiDAR sensing was selected for this project.

# Discussion - Odometry System Development

## Encoder-Based Odometry Development

The first major objective of the project was the development of an encoder-based odometry system capable of estimating robot displacement and travelled distance. Odometry forms the foundation of many mobile robot localisation systems because it provides a direct estimate of movement using wheel rotation measurements.

The robotic platform utilised quadrature wheel encoders mounted on each drive motor. Encoder pulses generated during wheel rotation were measured by the Arduino Uno and converted into wheel displacement values. These measurements were then used to estimate robot motion.

The distance travelled by each wheel was calculated using:

Distance = (Encoder Counts / Counts per Revolution) × Wheel Circumference

where:

- Encoder Counts represents the measured encoder pulses.

- Counts per Revolution (CPR) represents the number of pulses generated for one complete wheel revolution.

- Wheel Circumference is determined from the wheel diameter.

The wheel circumference was calculated as:

**C = πD**

where D is the wheel diameter.

For the robot platform used in this project, encoder measurements were continuously updated and processed to estimate robot displacement during motion.

![Figure 4.1.1 Encoder Integration and Testing Setup](/images/projects/mdetect/image20.png)

![Figure 4.1.2 Encoder Pulse Output from Oscilloscope](/images/projects/mdetect/image21.png)

Initial testing confirmed that encoder counts increased proportionally with wheel rotation, allowing reliable estimation of travelled distance. However, experimental results revealed several challenges including wheel slip, uneven motor performance, and accumulated odometry errors. These issues became more apparent during longer navigation tasks.

Despite these limitations, encoder odometry successfully provided real-time displacement information and established the foundation for subsequent localisation development.

![Figure 4.1.3 Print data of IMU MPU6050 Output](/images/projects/mdetect/image22.png)

IMU-only localization and Encoder was not stable enough for long-duration operation Combining sensors required careful synchronization

## Position Estimation Using Odometry

Following successful encoder integration, odometry equations were implemented to estimate robot displacement in the local coordinate frame.

The robotic platform used four-wheel encoders. Individual wheel displacement values were represented as:

d1, d2, d3 and d4

The local forward displacement was calculated as:

localY = (d1 + d2 + d3 + d4) / 4

The local lateral displacement was calculated as:

localX = (-d1 + d2 + d3 - d4) / 4

These equations allowed robot motion to be estimated relative to its current orientation.

The implementation enabled real-time calculation of robot movement in both forward and lateral directions. Position estimates were updated continuously as encoder data became available.

Experimental testing demonstrated that the odometry system could successfully track robot movement over short distances. However, accumulated errors became increasingly visible as travelled distance increased.

Sources of error included:

- Wheel slip

- Uneven floor surfaces

- Encoder measurement noise

- Mechanical tolerances

- Motor speed variations

As a result, additional sensing was required to improve localisation accuracy.

## IMU Integration and Heading Estimation

To improve localisation performance, an MPU6050 Inertial Measurement Unit (IMU) was integrated into the robot platform.

The MPU6050 combines a three-axis accelerometer and a three-axis gyroscope, enabling measurement of robot orientation and rotational movement.

The primary objective of IMU integration was to obtain heading information for use in coordinate transformation and position estimation.

![Figure 4.3.1 MPU6050 Wiring and Integration](/images/projects/mdetect/image23.png)

The sensor was calibrated during system startup to reduce gyroscope bias and improve heading accuracy.

Following calibration, heading values were continuously transmitted to the localisation algorithm.

The heading angle was represented as theta θ, where θ corresponds to the robot orientation relative to the global coordinate frame.

The addition of heading information enabled transformation of local robot movement into world coordinates.

![Figure 4.3.2 MPU6050 Heading Output](/images/projects/mdetect/image24.png)

Experimental testing confirmed that the IMU could reliably estimate heading changes during robot motion. This provided a significant improvement over encoder-only localisation approaches.

## World Frame Position Estimation

Once encoder displacement and heading information were available, coordinate transformation equations were implemented to estimate robot position within a global coordinate system.

The transformation from local coordinates to world coordinates was performed using:

worldX = localX cos(θ) − localY sin(θ)

worldY = localX sin(θ) + localY cos(θ)

The updated robot position was then calculated as:

Xnew = Xold + worldX

Ynew = Yold + worldY

These equations allowed robot movement to be represented in a fixed world coordinate frame rather than relative to the robot itself.

The implementation enabled real-time estimation of:

- X position

- Y position

- Heading angle (θ)

Position estimates were displayed through the monitoring system and updated continuously during robot operation.

Experimental results demonstrated that the robot could successfully estimate its position while navigating predefined paths.

![Figure 4.4.1 Real-Time Position Output](/images/projects/mdetect/image24.png)

The ability to estimate position within a world coordinate frame represented a major milestone in the project because it enabled waypoint navigation and future autonomous navigation capabilities.

## Localisation Performance Evaluation

The performance of the localisation system was evaluated through a series of controlled movement experiments. The robot was instructed to travel predefined distances while position estimates were monitored and recorded.

Results showed that:

- Encoder odometry provided reliable short-distance displacement measurements

- IMU heading estimation significantly improved orientation tracking

- Position estimation successfully produced real-time X and Y coordinates

- Localisation accuracy was sufficient for waypoint navigation experiments

However, several limitations were observed.

The primary limitation was accumulated odometry error. Small measurement inaccuracies in encoder counts and heading estimation gradually increased over time, causing deviation between estimated and actual robot position.

Additional factors contributing to localisation error included:

- Wheel slip during acceleration

- Uneven motor speeds

- Sensor noise

- Surface irregularities

These findings highlighted the need for additional filtering and motion correction techniques, which are discussed in the following chapter.

# Discussion – System Challenges and Performance Improvements

## Challenge 1: Motion Stability

Following the implementation of encoder-based odometry and IMU localisation, several motion tests were conducted to evaluate the robot's navigation performance. During these experiments, it was observed that the robot consistently deviated from the intended straight-line path.

Although identical motor commands were issued to all four motors, the robot gradually drifted to one side during forward motion. This behaviour reduced localisation accuracy and negatively affected waypoint navigation performance.

![Figure 5.1.1 Robot Drift During Straight-Line Motion](/images/projects/mdetect/image25.png)

Further investigation revealed that one motor produced slightly higher rotational speed than the remaining motors. As a result, unequal wheel velocities generated unintended changes in robot heading.

The issue became more significant as travelled distance increased because even small deviations accumulated over time and introduced errors into the odometry calculations.

To address this problem, encoder-based motor scaling was implemented. Experimental testing was performed to determine correction factors that compensated for motor speed variations.

The scaling process involved comparing encoder counts generated by each motor under identical operating conditions and applying correction coefficients to achieve more balanced wheel motion.

![Figure 5.1.2 Encoder Scaling Calibration Process](/images/projects/mdetect/image26.png)

Following implementation of motor scaling, the robot demonstrated significantly improved straight-line motion. Drift was reduced and overall localisation performance improved.

The results demonstrated that motor scaling provided an effective solution for compensating mechanical variations between motors and improving navigation accuracy.

## Challenge 2: Encoder Noise and Error Accumulation

Although encoder odometry successfully estimated robot displacement, testing revealed fluctuations in encoder measurements that affected localisation accuracy.

Several sources of measurement uncertainty were identified:

- Wheel vibration

- Surface irregularities

- Electrical interference

- Encoder pulse variations

- Mechanical tolerances

These effects produced noisy encoder measurements that occasionally resulted in sudden variations in calculated displacement.

As robot movement continued, small encoder errors accumulated and gradually increased localisation uncertainty.

To reduce measurement noise, Kalman filtering techniques were investigated and implemented.

The Kalman filter is a recursive estimation algorithm that combines prediction and measurement information to estimate the most probable system state.

For this project, the Kalman filter was applied to encoder measurements prior to position estimation calculations.

The filtering process reduced measurement fluctuations while preserving overall system responsiveness.

![Figure 5.2.1 Kalman Filter Algorithm (Borenstein and Liqiang Feng, 1996)](/images/projects/mdetect/image27.png)

Experimental results demonstrated a noticeable improvement in measurement stability.

The filtered encoder output exhibited:

- Reduced noise

- Smoother displacement measurements

- More stable localisation estimates

- Improved repeatability

Encoder Data before Kalman Filtering:

![Figure 5.2.2 Encoder Data before Kalman Filtering](/images/projects/mdetect/image28.png)

Encoder Data after Kalman Filtering:

![Figure 5.2.3 Encoder Data After Kalman Filtering](/images/projects/mdetect/image29.png)

The implementation of Kalman filtering contributed significantly to improving localisation performance and reducing accumulated odometry errors.

## Challenge 3: Accurate Motion Control

While localisation performance improved through odometry and filtering techniques, additional challenges were encountered during navigation experiments.

Repeated testing showed that the robot occasionally drifted during forward movement and experienced inaccuracies when executing turning manoeuvres.

Heading errors accumulated after multiple turns, causing deviations from the intended path.

The problem was influenced by several factors:

- Unequal motor performance

- Encoder measurement variations

- Surface friction differences

- Dynamic changes in robot motion

To improve motion accuracy, PID control techniques were implemented.

PID controllers continuously calculate the difference between a desired value and a measured value and generate corrective actions to minimise error.

Two PID control strategies were investigated:

**Encoder-Based PID Control**

Encoder feedback was used to maintain consistent wheel speeds and improve straight-line motion.

The controller continuously compared measured wheel motion with desired motion and adjusted motor outputs accordingly.

**IMU-Based Heading Control**

Heading information obtained from the MPU6050 was used to maintain the desired robot orientation.

When heading deviations occurred, corrective motor commands were generated to minimise orientation errors.

![A graph with a line drawn on it AI-generated content may be incorrect.](/images/projects/mdetect/image30.jpeg)

The combined use of encoder feedback and IMU heading information improved overall navigation performance.

Experimental observations indicated:

- Improved straight-line motion

- Reduced heading drift

- Better turning consistency

- More reliable waypoint navigation

![Figure 5.3.1 Navigation Performance After PID Implementation](/images/projects/mdetect/image31.png)

Although perfect accuracy was not achieved, the PID controllers significantly improved robot stability and navigation reliability.

## Challenge 4: System Integration and Resource Constraints

As additional functionality was added to the robot platform, limitations of the Arduino Uno became increasingly apparent.

The localisation system, encoder processing, IMU integration, PID control, and communication tasks collectively utilised a significant proportion of the available memory and processing resources.

During development, occasional system instability was observed, including:

- Unexpected freezes

- Controller communication issues

- Increased processing load

- Difficulty integrating additional sensing modules

The integration of a COIN-D6 LiDAR sensor introduced additional computational requirements that exceeded the practical capabilities of the Arduino Uno.

To address these limitations, a Raspberry Pi 4 was introduced into the system architecture.

The revised architecture separated low-level and high-level processing tasks.

**Arduino Responsibilities**

- Encoder processing

- IMU reading

- Motor control

- PID control

- Position estimation

**Raspberry Pi Responsibilities**

- LiDAR processing

- Data logging

- Monitoring console

- Obstacle detection

- Path planning

This architecture improved system scalability and allowed more advanced navigation functions to be developed without compromising real-time motor control performance.

The separation of processing tasks also simplified software development and provided a foundation for future expansion of the system.

# Results and Discussion – LiDAR Navigation System

## Raspberry Pi High-Level Control Development

As the project progressed, additional sensing and navigation capabilities were required beyond the processing capacity of the Arduino Uno. While the Arduino was suitable for encoder processing, IMU integration, and motor control, the implementation of LiDAR sensing and advanced navigation functions required a more powerful processing platform.

A Raspberry Pi 4 was therefore integrated into the robotic platform to perform high-level processing tasks. The Raspberry Pi was responsible for LiDAR communication, obstacle detection, monitoring functions, data logging, and future navigation algorithms.

Serial communication was established between the Arduino Uno and Raspberry Pi to enable real-time data exchange. The Arduino continuously transmitted robot localisation information, including position and heading estimates, while the Raspberry Pi performed environmental sensing and navigation functions.

The communication system transmitted:

- X position

- Y position

- Heading angle

- Waypoint information

- Navigation status messages

The implemented detection parameters included:

- Front obstacle detection zone

- Detection cone angle

When an obstacle was detected inside the safety region, the Raspberry Pi automatically generated an emergency stop command and transmitted it to the Arduino controller.

- Collision prevention

A\* is a graph-based search algorithm widely used in robotics for navigation and route optimisation.

The algorithm evaluates possible paths and selects a route with minimum cost while avoiding obstacles.

# Summary and Conclusions

## Summary

The objective of this project was to develop a GPS-free localisation and navigation platform for a mobile robotic system operating in underground and confined environments. The project was undertaken in collaboration with mDetect Pty Ltd to support future deployment of autonomous muon detector systems.

The project investigated the use of multiple sensing and navigation technologies including wheel encoders, an MPU6050 Inertial Measurement Unit (IMU), PID control, Raspberry Pi processing, and COIN-D6 LiDAR sensing.

The first stage of the project focused on encoder-based odometry. Quadrature wheel encoders were successfully integrated into the robotic platform and used to estimate wheel displacement and travelled distance. Real-time odometry calculations enabled continuous estimation of robot movement and formed the foundation of the localisation system.

The second stage involved integration of the MPU6050 IMU. Heading information obtained from the IMU was combined with encoder displacement measurements to estimate robot position within a two-dimensional coordinate system. The implementation enabled real-time estimation of X position, Y position, and heading angle (θ), satisfying the second project deliverable.

Several technical challenges were encountered throughout development. Motion instability caused by unequal motor performance resulted in localisation inaccuracies during straight-line motion. Encoder noise and accumulated measurement errors further reduced navigation performance. These issues were addressed through motor scaling, Kalman filtering techniques, and PID control strategies. The implemented solutions improved motion stability, reduced measurement noise, and enhanced navigation reliability.

As the project progressed, a Raspberry Pi 4 was integrated to provide additional computational capability for advanced sensing and navigation functions. Serial communication was established between the Arduino Uno and Raspberry Pi, enabling real-time transmission of localisation data.

The final stage of the project involved integration of a COIN-D6 Time-of-Flight LiDAR sensor. LiDAR processing software was developed using Python and enabled real-time environmental sensing, obstacle detection, monitoring, and data logging. Additional navigation features including waypoint management, emergency stop functionality, path planning concepts, and lightweight SLAM foundations were also investigated.

Overall, the project successfully combined multiple sensing technologies into a unified robotic localisation and navigation platform suitable for operation in GPS-denied environments.

## Conclusions

The results obtained throughout this project demonstrate that reliable GPS-free localisation can be achieved through the integration of encoder odometry, IMU heading estimation, and LiDAR sensing.

The developed system successfully achieved the primary project objectives:

- Encoder-based odometry was implemented for displacement estimation.

- IMU integration enabled heading estimation and orientation tracking.

- Real-time estimation of X, Y, and θ was achieved.

- Motion stability was improved through motor scaling and PID control.

- Raspberry Pi and Arduino communication was successfully established.

- COIN-D6 LiDAR integration enabled obstacle detection and environmental sensing.

- Advanced navigation concepts including waypoint management and path planning were investigated.

The implementation of a dual-controller architecture proved particularly beneficial. Low-level control functions were efficiently managed by the Arduino Uno, while high-level sensing and navigation functions were performed by the Raspberry Pi. This separation improved system scalability and supported future development of more advanced autonomous capabilities.

The project successfully transformed the robotic platform from a basic motion control system into a semi-autonomous navigation platform capable of localisation, environmental sensing, and obstacle detection.

These outcomes provide a strong foundation for future deployment of robotic sensing systems within underground and tunnel environments.

## Project Limitations

Although the project achieved its primary objectives, several limitations were identified. First, encoder odometry remains susceptible to accumulated errors caused by wheel slip, uneven surfaces, and mechanical tolerances. While filtering and motion correction techniques improved performance, localisation drift was still observed during extended operation.

Second, IMU measurements remain affected by sensor drift and calibration uncertainty. Long-term heading estimation accuracy may therefore decrease during prolonged operation.

Third, the obstacle detection system was primarily designed for safety monitoring rather than full autonomous obstacle avoidance. Additional development is required to achieve robust navigation in complex environments.

Finally, full SLAM implementation was outside the scope of this project. Although the system provides the foundation for future mapping and localisation development, complete autonomous mapping capability has not yet been achieved.

## Recommendations for Future Work

Several opportunities exist for future development and improvement of the robotic platform.

**Full SLAM Implementation**

Future work should focus on implementing a complete Simultaneous Localisation and Mapping (SLAM) system. SLAM would enable the robot to generate maps of unknown environments while simultaneously estimating its position within those maps.

**Extended Kalman Filter (EKF)**

The current localisation system could be enhanced through implementation of an Extended Kalman Filter. Sensor fusion techniques combining encoder, IMU, and LiDAR measurements would improve localisation accuracy and reduce accumulated errors.

**Autonomous Obstacle Avoidance**

Future development should include fully autonomous obstacle avoidance and dynamic route replanning capabilities. Integration of obstacle avoidance with path planning algorithms would improve navigation reliability in complex environments.

**ROS2 Integration**

The Robot Operating System (ROS2) could be integrated to provide a more scalable software architecture and access to advanced robotics tools and libraries.

**Real-Time Mapping and Visualisation**

Additional work could focus on developing real-time environment mapping and visualisation capabilities using LiDAR data. This would provide improved situational awareness for both autonomous and teleoperated operation.

**Muon Detector Integration**

The final stage of development should involve integration of the muon detector payload with the localisation platform. This would enable precise association of muon measurements with robot position and orientation, supporting the objectives of mDetect Pty Ltd.

**Field Testing**

Future testing should be conducted in representative tunnel and underground environments to evaluate system performance under realistic operating conditions.

## Final Remarks

This project successfully demonstrated the development of a GPS-free robotic localisation and navigation platform using encoder odometry, IMU heading estimation, Raspberry Pi processing, and LiDAR sensing technologies.

The system achieved reliable position estimation, obstacle detection, and monitoring capabilities while establishing a framework for future autonomous navigation and underground sensing applications.

The knowledge gained throughout this project has contributed significantly to the understanding of robotic localisation, sensor integration, embedded systems, and autonomous navigation technologies. The developed platform provides a valuable foundation for future research and industrial deployment within GPS-denied environments.
